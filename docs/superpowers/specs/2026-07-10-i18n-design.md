# Diseño: soporte de idioma (es/en) en todo el CLI

Fecha: 2026-07-10. Estado: aprobado por el usuario (enfoque A: módulo i18n con
patrones compartidos + fragmentos bilingües por pack).

## Problema

Todo el texto — contenido generado, prompts, mensajes del CLI — está
hardcodeado en español. El paquete tiene audiencia global en npm: para un
equipo no hispanohablante el CLAUDE.md generado es inservible.

## Decisiones (tomadas con el usuario)

1. **Selección de idioma**: auto-detección del locale del sistema
   (`Intl.DateTimeFormat().resolvedOptions().locale`; prefijo `es` → español,
   cualquier otro → inglés, fallback inglés ante error). El flag `--lang es|en`
   fuerza el idioma; un valor inválido produce error + ayuda y exit 1.
2. **Alcance**: se traduce TODO — contenido generado (secciones de stack,
   secciones de RepoFacts, prompts, fallback), mensajes interactivos,
   `--help`, avisos y el prompt de pulido del llm-bridge.
3. **Arquitectura**: los patrones de texto compartidos entre packs viven una
   vez en `core/i18n.ts` como funciones bilingües; cada pack conserva solo sus
   fragmentos específicos en una tabla `{ es, en }`. Nada de traducción
   post-render ni duplicación completa por pack.

## `core/i18n.ts`

```typescript
export type Lang = "es" | "en";

export function detectLang(): Lang; // Intl locale, "es*" → "es", resto → "en", catch → "en"
```

Constructores compartidos (cada uno con su plantilla es/en):

| Función | es (actual) | en |
|---|---|---|
| `summarySentence(lang, language, framework?, parenthetical?)` | `Proyecto {language} con {fw} ({paren}).` | `{language} project using {fw} ({paren}).` |
| `runTestsConvention(lang, cmd)` | `Ejecuta los tests con {cmd} antes de terminar una tarea.` | `Run the tests with {cmd} before finishing a task.` |
| `reviewBody(lang, focus, framework)` | `Revisa el diff actual buscando bugs, {focus} y desviaciones de las convenciones de {framework}. Señala solo problemas concretos con línea de archivo.` | `Review the current diff looking for bugs, {focus} and deviations from {framework} conventions. Point out only concrete issues with file and line.` |
| `refactorBody(lang, extra?)` | `Propón refactors que reduzcan duplicación y mejoren la legibilidad sin cambiar comportamiento observable.{extra}` | `Propose refactors that reduce duplication and improve readability without changing observable behavior.{extra}` |
| `testingBody(lang, runner)` | `Escribe tests con {runner} para el código señalado. Cubre el camino feliz y al menos un caso límite.` | `Write tests with {runner} for the highlighted code. Cover the happy path and at least one edge case.` |
| `unknownRunnerLabel(lang)` | `el test runner del proyecto` | `the project's test runner` |
| `unknownFrameworkLabel(lang)` | `el framework del proyecto` | `the project's framework` |

Textos de interfaz (objeto `UI: Record<Lang, {...}>`): cabecera de documento
("Generado por agent-rules-init…" / "Generated by agent-rules-init…"),
títulos de secciones de facts ("Comandos del repo" / "Repo commands",
"Estructura" / "Structure", "Lo que ejecuta CI (GitHub Actions)" / "What CI
runs (GitHub Actions)"), nota "…y N más{ en X}" / "…and N more{ in X}",
fallback sin stack, usage de `--help`, intro/outro/avisos del CLI, aviso de
TTY no interactiva, mensajes del pulido, y la pregunta interactiva:
`No se pudo determinar {campo} para {lenguaje}. ¿Cuál usáis?` /
`Couldn't determine {field} for {language}. Which one do you use?` con
etiquetas de campo legibles (`el framework`/`the framework`, `el test
runner`/`the test runner`, `el linter`/`the linter`, `el gestor de
paquetes`/`the package manager`) — esto elimina de paso el nombre de campo
crudo en inglés ("el testRunner") de las preguntas actuales.

Las anotaciones de directorios de RepoFacts (`DIR_NOTES`) pasan a ser
bilingües: `Record<string, Record<Lang, string>>`.

## Cambios de contrato

- `Pack.rules(detection, lang)` y `Pack.promptTemplates(detection, lang)`
  (parámetro nuevo obligatorio). Los 15 packs se refactorizan: patrones
  compartidos → llamadas a i18n; fragmentos específicos (guía de estilo,
  cláusula de foco del review, notas de arquitectura, convención de
  dependencias) → tabla local `TEXTS: Record<Lang, {...}>`. La traducción al
  inglés de cada fragmento se hace pack a pack a partir del texto español
  existente en el propio archivo.
- Los títulos de PromptTemplate ("Code Review (Python)", "Refactor (Go)"…)
  ya son neutros y no cambian.
- `buildRepoFacts(signals, lang)` — por las anotaciones de estructura.
- `renderClaudeMd/renderAgentsMd/renderCopilotInstructions(entries, facts, lang)`
  — `lang` obligatorio (igual que en los packs): sin defaults silenciosos en
  producción. Los tests existentes se actualizan para pasarlo explícito.
  `facts` sigue siendo opcional.
- `runCli(rootPath, options)` gana `lang?: Lang`; si no se pasa, `detectLang()`.
- `resolveCliAction(argv)` reconoce `--lang es`, `--lang=es` (y `en`);
  devuelve `{ kind: "run", lang? }`; valor inválido → `{ kind: "invalid-lang", value }`
  que imprime error + usage y exit 1. `--help`/`--version` mantienen prioridad.
- `polishWithAssistant(assistant, content, execFn, lang)` — prompt bilingüe.
- `docs/writing-a-pack.md` se actualiza con el contrato nuevo.

## Comportamiento

- `npx agent-rules-init` en un sistema con locale `es-*` → todo en español;
  en cualquier otro → todo en inglés. `--lang` fuerza. El idioma afecta a los
  archivos generados y a todos los mensajes de la ejecución.
- El README pasa a inglés como idioma principal con una sección breve en
  español (parte del punto "publicación": bump a 0.3.0).

## Testing

- Unit: `detectLang` (con mock de `Intl`), constructores i18n en ambos
  idiomas, `resolveCliAction` con `--lang` (válido, `=`, inválido).
- Packs: los tests existentes pasan `"es"` y mantienen sus asserts; se añade
  al menos un assert en inglés por pack (summary o template).
- Templates/facts: secciones con títulos en ambos idiomas.
- e2e en `cli.test.ts`: `runCli(..., { lang: "en" })` genera contenido inglés
  (sin rastro de "Proyecto"/"Comandos del repo").
- Validación manual con el binario: `--lang en` y `--lang es` sobre un repo
  sintético; `--lang xx` → exit 1.
